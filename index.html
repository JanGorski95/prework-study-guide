HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • A link element creates the connection to the external file - The rel attribute specifies the relationship between the current document and the linked document resource.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The period preceding card designates this as a class selector
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • To apply a rule to all the elements visible on the page, we use a special wild card symbol, or *
  • Do Not Repeat Yourself (DRY)

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: dd the changed files in our working directory to the staging area of Git.
  • git commit -m "description": commit all changes to our repository
  • Git Flow: git add -A / git commit -m "comment" / git pull origin main / git push origin branch-name / "Pull Requests" / git checkout main / git pull origin main

JavaScript

  • Strings need to be contained within quotation marks
  • A function's name is followed by parentheses ()
  • You must call a function by name in order to execute it - can call it as many times as we need
  • Control flow is the order in which a computer executes code in a script.
  • To randomize our selection, we need to use a special Math function
